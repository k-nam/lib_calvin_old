S009:辞書順最小

長さ n の数列 S = {S_1, S_2, ..., S_n} を並べ替えることを考えてみましょう。
ただし，並べ替えに使うことができるのは m 種類ある長さ n の置換だけです。
ここで，長さ n の置換 P とは，{1, 2, ..., n}を並べ替えてできる 数列 P = {p_1, p_2, ..., p_n}のことを言います。

数列 S に置換 P を適用することで新たな数列 T を得ることができます。
具体的には，S_1 を T_{p_1} に，S_2 を T_{p_2} に，..., S_n を T_{p_n} にすることで数列 T を得ます。

今 m 種類の置換をそれぞれ 1 回以下ずつ使って数列 S を並べ替えることを考えます。置換を適用する順序は任意で構いません。
そのうえで、得られる数列の中で辞書順比較したときに最小となるものを求めるプログラムを作成してください。

長さ n の数列 A と B を辞書順比較したとき、A が B よりも小さいとは、次が成り立つことをいいます．
・ある整数 i (1 ≦ i ≦ n) に関して、1 ≦ j ≦ i-1 を満たすどの整数 j についてもA_j = B_j が成り立ち、かつ A_i < B_i が成り立つ。
・ここで、 A_j, B_j はそれぞれ A, B の j 番目の要素を表す。

例1の最初の置換の様子は次のようになっています。
P_1 の第 1 項は 1 → S の第 1 項は 3 →置換後の数列の第 1 項は 3
P_1 の第 2 項は 3 → S の第 2 項は 1 →置換後の数列の第 3 項は 1
P_1 の第 3 項は 2 → S の第 3 項は 2 →置換後の数列の第 2 項は 2
最終的に数列 {3, 2, 1} となります。

評価ポイント
10個のテストケースを入力し、正答数と解答の提出までに要した時間を測定し得点が決まります。
※提出いただいたコードは複数回実行され、一度の実行では1つのテストケースのみ入力
※制限時間を超えるとテストケースが通っても失格(0点)となります。
得点の計算方法：正解数得点(50点) ＋ 正解率×回答時間得点(40分以内で50点、80分で25点、2時間で0点と線形に点数が落ちます)
  1. 10個のテストケースで正しい出力がされるか評価 (50点)
  2. 解答の提出までに要した時間による評価 (50点)
入力される値
入力は次のフォーマットで与えられます。
n
S_1 S_2 ... S_n
m
p_{1,1} p_{1,2} ... p_{1,n}
p_{2,1} p_{2,2} ... p_{2,n}
...
p_{m,1} p_{m,2} ... p_{m,n}
1 行目には数列の長さ n が 、2 行目には長さ n の数列 S が与えられます。
3 行目には置換の種類数 m が、 続く m 行には m 種類の長さ n の置換が与えられます。 p_{i,j} は i 種類目の置換の j 番目の値です。
入力される値は全て整数です。

文字列は標準入力から渡されます。標準入力からの値取得方法はこちらをご確認ください
期待する出力
数列 S に m 種類の置換をそれぞれ 1 回以下ずつ適用したとき、 得られる数列のうち辞書順で最小のものを以下のフォーマットで出力してください。
ここで、出力する数列を T と仮定します。
T_1 T_2 ... T_n
最後は改行し、余計な文字、空行を含んではいけません。
条件
すべてのテストケースにおいて、以下の条件を満たします。

・3 ≦ n ≦ 10
・1 ≦ S_i ≦ n
・i ≠ j ならば S_i ≠ S_j
・1 ≦ m ≦ 5
・1 ≦ p_{i,j} ≦ n
・j ≠ k ならば p_{i,j} ≠ p_{i,k}
